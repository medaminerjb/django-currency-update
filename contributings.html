Contributing¶

We welcome contributions to the django-currency-updater library!
If you’d like to contribute, follow these steps:

Reporting Bugs¶

If you find a bug, please open an issue on GitHub with a description of the problem and steps to reproduce it.

Submitting a Pull Request¶

  1. Fork the repository on GitHub.

  2. Clone your fork locally.

  3. Create a new branch for your feature or bugfix.

  4. Make your changes, and add tests if necessary.

  5. Commit your changes and push them to your fork.

  6. Submit a pull request to the main repository.

We will review your pull request and may ask for additional changes or clarifications.

Code Style¶

Please follow the PEP 8 style guide for Python code. We also use black to format the code, so ensure that your changes are formatted before submitting a pull request.

We also require that new features or bug fixes include corresponding tests to ensure the stability of the library.
